1. حسابات المستخدمين، الصلاحيات والأمور المالية
1. جدول بيانات الطلاب (student): يخزن الملف الشخصي والأكاديمي الشامل لكل طالب، ويرتبط بالحالة المالية وحسابات أولياء الأمور.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي والمعرف الفريد لكل طالب.
full_name VARCHAR(150) Standard الاسم الكامل للطالب المستخدم في القوائم والشهادات.
student_code VARCHAR(50) UNIQUE Standard الرقم التسلسلي الفريد والخاص بالطالب داخل المنصة.
academic_level VARCHAR(50) Standard الصف أو المستوى الدراسي الحالي للطالب.
parent_id INT Standard رقم هوية ولي الامر.
parent_phone VARCHAR(20) Standard رقم التواصل الأساسي لولي الأمر.
parent_backup_phone VARCHAR(20) Standard رقم الهاتف الاحتياطي لولي الأمر في حالات الطوارئ.
account_status VARCHAR(20) Standard حالة حساب الطالب (نشط أو معطل إدارياً) في حال لم يتم دفع الاقساط يتم تعطيل الحساب مؤقت.
2. جدول الرسوم والمطالبات الشهرية (student_monthly_fees): إدارة وتتبع الشؤون المالية والمدفوعات لكل طالب لضمان انتظام الأقساط، وحساب المتأخرات تلقائياً.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي والسجل المالي الفرعي لكل عملية.
student_id INT Foreign Key يربط الحركة بالطالب المستحق عليه القسط المالي.
academic_level VARCHAR(50) Standard المرحلة الدراسية للطالب وقت المطالبة (لتحديد قيمة القسط تلقائياً).
billing_month VARCHAR(20) Standard الشهر والسنة المستحق فيها القسط (مثل: 06-2026).
monthly_amount DECIMAL(10,2) Standard قيمة القسط الشهري الإجمالي المطلوب دفعه (مثال: 500.00).
paid_amount DECIMAL(10,2) DEFAULT 0.00 Standard إجمالي المبالغ التي تم سدادها وتراكمها فعلياً من قسط هذا الشهر.
payment_method VARCHAR(50) NULL Standard طريقة آخر دفعة تمت (نقداً، تحويل بنكي، محفظة إلكترونية).
payment_date TIMESTAMP NULL Standard تاريخ ووقت تسجيل آخر عملية دفع بدقة متناهية.
notes TEXT NULL Standard مربع الملاحظات الإدارية والمالية الخاصة بالحركة المكتوبة بالفورم.
3. جدول المعلمين (teachers): إدارة الحسابات الشخصية والمهنية لأعضاء الهيئة التدريسية وتفاصيل موادهم.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي والمعرف الفريد للمعلم.
full_name VARCHAR(150) Standard الاسم الكامل للأستاذ الظاهر في واجهات المناهج.
teacher_code VARCHAR(50) UNIQUE Standard الرقم الوظيفي التسلسلي الخاص بالمعلم.
phone_number VARCHAR(20) Standard رقم هاتف المعلم للتواصل الإداري والتنسيق.
subject VARCHAR(100) Standard المادة الأساسية الموكلة للأستاذ لتدريسها.
role_id INT Foreign Key يربط المعلم بجدول الصلاحيات والأدوار المسموحة له.
4. جدول الأدوار والصلاحيات (roles_permissions): تطبيق نظام الحماية والتحكم بالصلاحيات القائم على الأدوار (RBAC) للمعلمين والمشرفين.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي لتعريف الدور الصلاحياتي.
role_name VARCHAR(50) Standard اسم الدور الممنوح (معلم، مشرف، مدير، إلخ).
can_manage_attendance BOOLEAN Standard مؤشر منح أو حجب صلاحية أخذ الحضور والغياب.
can_upload_files BOOLEAN Standard صلاحية رفع ملفات المناهج والمحاضرات للطلاب.
can_create_exams BOOLEAN Standard صلاحية إنشاء وإعداد امتحانات إلكترونية من البنك.
can_edit_grades BOOLEAN Standard صلاحية رصد وتعديل علامات ودرجات الطلاب.
can_reply_messages BOOLEAN Standard صلاحية فتح غرف الشات والرد على رسائل أولياء الأمور.
5. جدول صفوف المعلمين (teacher_academic_levels): جدول وسيط لربط المعلم بالصفوف الأكاديمية المختلفة التي يقوم بتدريسها (كسر علاقة M:N).
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المعرف الفريد لسجل الربط.
teacher_id INT Foreign Key يرتبط بالمعلم المعني بالصف الدراسي.
academic_level VARCHAR(50) Standard الصف الأكاديمي المسند تدريسه للمعلم.
section_name VARCHAR(50) Standard الشعبة، مثل: شعبة (أ).
6. جدول المستخدمين الموحد (users): إدارة بيانات تسجيل الدخول وتوثيق الهوية الرقمية لكافة حسابات أطراف المنصة.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
identity_number VARCHAR(50) Primary Key رقم الهوية الشخصي الفريد للمستخدم كمفتاح دخول.
name VARCHAR(150) Standard الاسم المستخدم عند تسجيل الدخول للنظام.
password VARCHAR(255) Standard كلمة المرور السرية المخزنة مشفرة ومؤمنة بالكامل.
role VARCHAR(30) Standard فئة وصلاحية الحساب لتوجيه الواجهات (admin/student/teacher).
2. منظومة الدروس والمحاضرات الرقمية والتفاعلية
7. جدول الدروس والمواد التعليمية (lessons): مستودع المحاضرات الرقمية والمواد الإثرائية المرفوعة من المعلمين للطلاب.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي للدرس أو المحاضرة.
teacher_id INT Foreign Key يربط الدرس بالمعلم الناشر والمعد للمادة الشارحة.
subject_name VARCHAR(100) Standard اسم المادة التعليمية التابع لها المحتوى.
title VARCHAR(200) Standard عنوان المحاضرة (مثل: الكيمياء الأساسية وعناصر الجدول الدوري).
file_type VARCHAR(50) Standard نوع الملف المرفق لدعم الواجهة (فيديو شرح، ملف PDF).
file_url VARCHAR(255) Standard المسار السحابي أو رابط تخزين الملف على خوادم المنصة.
views_count INT DEFAULT 0 Standard عداد إجمالي عدد المشاهدات والزيارات للدرس من الطلاب.
downloads_count INT DEFAULT 0 Standard عداد لتتبع عدد مرات تحميل الملف النصي المرفق.
created_at TIMESTAMP Standard تاريخ ووقت نشر وإتاحة المحتوى على لوحة الطالب.
8. جدول شعب الدروس (lesson_sections): يربط الدروس بالشعب المستهدفة لضمان ظهور الدرس للفئة الدراسية المستهدفة فقط.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المعرف الفريد لحركة الربط للشعبة.
lesson_id INT Foreign Key يرتبط بالدرس المستهدف نشره لعدة شعب.
section_name VARCHAR(20) Standard اسم الشعبة المخول لطلابها مشاهدة الدرس (مثل: شعبة أ).
9. جدول الحضور والغياب اليومي (attendance_logs): تتبع ومتابعة انضباط الطلاب ورصد حالات حضورهم اليومية في الحصص المختلفة.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي لسجل رصد الانضباط اليومي.
student_id INT Foreign Key يربط الحركة بالطالب المرصود حضورياً.
teacher_id INT Foreign Key يرتبط بالمعلم الذي قام بمراجعة ورصد الحضور والغياب.
date DATE Standard التاريخ الفعلي والرسمي لليوم الدراسي الحاضر.
status VARCHAR(30) Standard حالة الانضباط المرصودة للطالب (حاضر - غائب - متأخر).
3. منظومة بنك الأسئلة، الاختبارات الذكية، ورصد النتائج الفورية
10. جدول الامتحانات (Exams): يمثل الغلاف والرأس الكلي للاختبار متضمناً الضوابط والوقت المقيد للطالب لإنهاء التقييم.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي للاختبار المولد من البنك.
teacher_id INT Foreign Key المعلم منشئ الامتحان والمسؤول الأول عن نشر تقييمه.
title VARCHAR(150) Standard عنوان الامتحان ومسمى المادة (مثل: اختبار الكيمياء النصفي).
academic_level_subject VARCHAR(150) Standard تخزين وتحديد المادة والصف الدراسي التابع له الامتحان.
duration_minutes INT Standard مدة الاختبار بالدقائق لضبط ساعة العداد التنازلي البرمجي.
total_questions INT Standard عدد الأسئلة الإجمالي التي سيتعرض لها الطالب في الامتحان.
created_at TIMESTAMP Standard وقت وتاريخ إصدار ونشر الامتحان للطلاب على المنصة.
11. جدول أسئلة الامتحانات الوسيط (Exam_Questions): يربط الاختبار بالأسئلة المسحوبة له من بنك الأسئلة لتمكين تدوير وإعادة استخدام الأسئلة.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key معرف فريد يمثل وجود سؤال معين بداخل امتحان معين.
exam_id INT Foreign Key المفتاح الأجنبي المرتبط بجدول الامتحانات الرأسية.
questions_bank_id INT Foreign Key المفتاح الأجنبي المرتبط بالسؤال الأصلي المستدعى من البنك.
12. جدول الأسئلة المركزي (questions_bank): مخزن الأسئلة الشامل للمنصة، المصنف حسب الصفوف والمواد ومستويات الصعوبة والأنواع.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي لكل سؤال بداخل مستودع الأسئلة.
teacher_id INT Foreign Key المعلم معد ومبتكر هذا السؤال البرمجي.
academic_level_subject VARCHAR(150) Standard تصنيف وتحديد الصف والمادة التابع لها السؤال لغايات الفلترة.
question_text TEXT Standard نص ومضمون السؤال التعليمي الظاهر للطلاب أثناء تقديم التقييم.
question_type VARCHAR(30) Standard نوع طبيعة السؤال البرمجية (الاختيار من متعدد mcq / صح وخطأ true_false).
difficulty_level VARCHAR(30) Standard مستوى الصعوبة المعتمد للمولد الذكي للامتحانات (easy/medium/hard).
13. جدول خيارات الأسئلة (question_options): يحتوي على الخيارات الأربعة المتاحة لأسئلة الاختيار من متعدد مع تحديد الإجابة النموذجية.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي التعريفي للخيار الفرعي للسؤال.
questions_bank_id INT Foreign Key يربط مجموعة الخيارات المتعددة بالسؤال الأم التابعة له في البنك.
option_text VARCHAR(255) Standard النص أو الحقل الرقمي المعروض كخيار للحل أمام الطالب.
is_correct BOOLEAN Standard علم منطقي (true/false) يعتمد الإجابة الصحيحة للتصحيح الفوري تلقائياً.
14. جدول تفاصيل إجابات الطلاب (student_exam_answers): يسجل بشكل مفصل خيارات الطلاب أثناء تأدية الامتحان، مما يتيح للمعلمين مراجعة آلية تفكير الطالب وتحليل الأخطاء الشائعة.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المعرف الفريد لحركة حل وضغط زر الطالب بداخل الامتحان.
student_id INT Foreign Key يرتبط بالطالب الفرد الذي يقوم بتقديم الإجابة حالياً.
exam_id INT Foreign Key رقم الامتحان الإجمالي المحلول داخله السؤال.
questions_bank_id INT Foreign Key معرف السؤال الدقيق الذي تتم الإجابة عليه الآن من الطالب.
selected_option_id INT Foreign Key رقم الخيار الدقيق والمحفوظ الذي ضغط عليه الطالب كحل للسؤال.
15. جدول نتائج وعلامات الطلاب (student_exam_results): الرصيد النهائي لعلامات الطلاب الكلية في الاختبارات وتوثيق طرق تقديمهم للامتحانات.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key رقم وثيقة الرصد الكلي للعلامة والدرجة النهائية للطالب.
student_id INT Foreign Key يربط العلامة النهائية الكلية بالطالب صاحب التقييم.
exam_id INT Foreign Key يربط النتيجة بالامتحان الإجمالي الذي تم رصده وتصحيحه.
score_obtained INT Standard الدرجة الرقمية الكلية المحصلة والمستحقة للطالب فور إنهاء الحل.
status VARCHAR(50) Standard حالة رصد النتيجة وتدقيقها (مصحح تلقائياً، تم رصده، غياب تلقائي).
submission_method VARCHAR(100) Standard طريقة تقديم الطالب للامتحان (حساب الطالب الإلكتروني أو لم يدخل).
4. منظومة الواجبات المنزلية، التكليفات والتسليمات المرفوعة
16. جدول الواجبات المنشورة (assignments): إدارة الواجبات المنزلية والمهام المحددة من قِبل المعلمين ومواعيد تسليمها وضوابط الدرجات المتاحة.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي والرقم التعريفي للواجب المنشور.
teacher_id INT Foreign Key المعلم صاحب المهمة والتكليف والمسؤول عن مراجعته وتصحيحه لاحقاً.
academic_level_subject VARCHAR(150) Standard تحديد وحفظ المادة والصف الدراسي الموجه له الواجب الدراسي.
title VARCHAR(200) Standard عنوان الواجب الرئيسي (مثل: واجب إعراب الجملة الفعلية والأفعال الخمسة).
due_date DATETIME Standard آخر موعد وتاريخ محدد وصارم متاح أمام الطالب لإرسال حل الواجب.
description TEXT Standard التوصيف والتوجيه النصي التفصيلي لمضمون وحل الواجب المطلوبة.
attachment_url VARCHAR(255) Standard رابط أو مسار ملف الأسئلة الاختياري المرفق مع الواجب من المعلم.
total_grade INT Standard درجة الواجب الإجمالية" (مثلاً: الواجب من 10 درجات أو 20 درجة).
created_at TIMESTAMP Standard وقت إطلاق ونشر المهمة على لوحة تحكم حسابات الطلاب المعنيين.
17. جدول تسليمات الواجبات (assignment_submissions): توثيق حلول الطلاب المرفوعة كملفات خارجية وتقييم ورصد المعلم لدرجاتها الفردية.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي لسجل حركة تسليم ملف حل الطالب.
assignment_id INT Foreign Key يربط حركة التسليم بالواجب الأصلي المطلوب تسليمه.
student_id INT Foreign Key يرتبط بالطالب الذي قام بحل ورفع ملف الواجب للمنصة.
submitted_file_url VARCHAR(255) Standard رابط ومسار ملف الحل المرفوع من الطالب (كراسة إجابة PDF أو صورة).
grade INT NULL Standard الدرجة والعلامة المستحقة التي يضعها المعلم يدوياً بعد مراجعة ملف الحل.
status VARCHAR(50) Standard حالة تصحيح التسليم الحالي لمتابعة الواجهات (لم يتم التصحيح، تم التصحيح).
submitted_at TIMESTAMP Standard الوقت والتاريخ الدقيق والآلي لقيام الطالب بضغط زر إرسال للحل.
5. منظومة المراسلة، الشات، الإشعارات السحابية وإدارة الأوفلاين بالمزامنة
18. جدول غرف المحادثات (chat_rooms): ينظم بيئة الدردشة والتواصل من خلال إنشاء قنوات تواصل فردية ومستقلة تجمع المعلم بالطالب.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي لتعريف قناة وغرفة التواصل الفردية.
teacher_id INT Foreign Key المعلم المشترك بداخل هذه المحادثة الفردية الحالية.
student_id INT Foreign Key الطالب (أو حساب ولي أمره المتابع) المشترك في نفس الغرفة.
created_at TIMESTAMP Standard تاريخ ووقت التأسيس الأولي لغرفة المحادثة والشات المباشر.
19. جدول الرسائل (messages): تخزن تفاصيل الرسائل النصية والملفات المتبادلة داخل غرف المحادثة وحالة قراءتها وتوقيتها الفعلي.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي لكل فقاعة ورسالة مبعوثة داخل الشات.
chat_room_id INT Foreign Key يربط الرسالة الحالية بغرفة المحادثة الشاملة التابعة لها.
sender_type VARCHAR(20) Standard تحديد جهة فئة المرسل لحقن وترتيب الرسالة في الواجهات (teacher/student).
message_text TEXT Standard المضمون والنص المكتوب والمتبادل بداخل الشات التعليمي المباشر.
attachment_url VARCHAR(255) NULL Standard رابط أو مسار ملف أو صورة مرفقة بشكل اختياري بداخل حزمة الرسالة.
is_read BOOLEAN DEFAULT FALSE Standard حقل منطقي لتفعيل علامة المقروئية والنقطة الزرقاء بواجهة التطبيق.
created_at TIMESTAMP Standard وقت إرسال الرسالة بدقة متناهية لغايات ترتيب خط الدردشة الزمني.
20. جدول الإعلانات - الإشعارات (Announcements): بث وتخزين التنبيهات الفورية والتحديثات الهامة للمستهدفين عند حدوث متغير تعليمي.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المفتاح الأساسي السحابي لكل إشعار وتنبيه منشور للمستخدمين.
user_id INT Standard الرقم التعريفي للمستخدم المستهدف ببث التنبيه الحالي (معلم/طالب).
title VARCHAR(150) Standard العنوان التنبيهي والمختصر الموجه (مثل: تم رصد علامة اختبار الرياضيات).
content TEXT Standard النص والمضمون الكامل والشارح للإشعار المنشور على شاشة التنبيهات.
action_url VARCHAR(255) Standard رابط انتقال سريع مدمج (Deep Link) ينقل المستخدم للشاشة المعنية عند النقر.
created_at TIMESTAMP Standard وقت وتاريخ إصدار وبث الإعلان والتنبيه السحابي للمستخدم.
is_read BOOLEAN DEFAULT FALSE Standard حقل منطقي لتحديد قراءة التنبيه لإخفاء علامة النقطة التنبيهية الحمراء.
💎 جدول العمليات المعلقة أوفلاين (Pending_Actions): قاعدة البيانات المحلية "الحصالة" داخل جهاز الطالب لحفظ عملياته مؤقتاً عند انقطاع الإنترنت ومزامنتها فوراً عند عودة الشبكة.
اسم الحقل نوع البيانات نوع المفتاح الوصف والوظيفة البرمجية
id INT AUTO_INCREMENT Primary Key المعرف الفريد لكل عملية مخزنة محلياً بجهاز الطالب بانتظار المزامنة.
student_id INT Foreign Key رقم كود الطالب الذي أجرى تلك الحركة والإنترنت مقطوع عن جهازه.
action_type VARCHAR(50) Standard نوع الإجراء المدخر محلياً لتفكيكه وفرزه (مثل: submit_exam أو send_message).
payload_json TEXT Standard حزمة البيانات المدخرة بصيغة نصية مكثفة JSON لرفعها فوراً لخوادم السيرفر الرئيسي.
timestamp TIMESTAMP Standard تاريخ ووقت تسجيل الحركة محلياً لفرز وترتيب واجهة طابور المزامنة المنتظر.
